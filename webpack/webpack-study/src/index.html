一、webpac 能做什么？
        1.webpack能够处理 JS 文件的互相依赖关系
        2.webpack能够处理 JS 的兼容问题，把高级的、浏览器不支持的语法，转为 低级的，浏览器能够正常识别的语法

        二、刚刚运行的命令格式： webpack 要打包的文件的路径 -o  打包好的输出文件的路径 --mode development
        如果改变 main.js 需要重新执行打包命令
        1.根目录创建webpack.config.js 项目webpack配置文件，
        2.配置好后，执行webpack命令，直接打包文件

        三、当我们在控制台，直接输入 webpack 命令，webpack进行以下步骤
        1.首先，webpack发现，我们并没有通过命令形式指定 入口 和 出口
        2.webpack 就回去项目的 根目录中，查找一个叫做'webpack.config.js'的配置文件
        3.当找到配置文件后，webpack 会去解析执行这个配置文件，当解析执行完配置文件后，就得到配置文件中，导出的配置对象
        4.当webpack拿到 配置对象 后，就拿到了指定的 入口 和 出口，然后就执行 打包构建

        四、使用webpack-dev-server 工具来实现，自动打包编译的功能
        1.运行npm i webpakc-dev-server -D 把这个工具安装到项目的本地开发依赖
        2.安装完毕后，工具用法和webpack命令用法完全一样
        3.由于，是在项目中，本地安装的  webpack-dev-server 所以无法把它当做脚本命令，在终端中直接运行，
        只有那些全局-g安装的工具，才能在终端中正常执行
        4.webpack-dev-server想要正常运行，必须在本地项目中安装webpack
        5.webpack-dev-server打包生成的bundle.js并没有存放到实际物理磁盘上，
        而是直接托管到了电脑内存中，所以电脑中找不到打包好的bundl.js

        webpack4.4中，直接采用 webpack --watch 即可监听文件，自动编译

        区别是：

        npm run dev 执行 webpack-der-server 监测到代码变化后，浏览器可以看到及时更新的效果，但是并没有自动打包修改的代码；
        webpack --watch 在监测到代码变化后自动打包修改的代码
        备注：这些加在命令后面
        　　--inline 修改代码后，浏览器自动刷新
        　　--inline --hot 修改代码后，浏览器不刷新，只更新修改的部分 ，在修改弹窗内容时候很有作用
        　　--profile 显示编译的具体耗时
        　　--process 显示编译的进度
